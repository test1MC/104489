Care Checklists Menu

The Care Checklists menu has a view sort option dropdown which determines what the main screen will show.

Depending on the selection from this screen, the resulting main screen will have differing results. Each of these will be detailed below.

All: Shows all available checklists in the system. This list could be lengthy. Other sort and search features are available to shorten the returned results.

Instances: Will display all instances of each checklist.

Recently Viewed: Will display the most recently viewed checklists by logged in UserID. This list can be sorted. It is updated immediately.

Templates: Shows all available Care Checklist templates within the system.

Top Level Menus

 

The top level menu options will vary depending on which Care Checklists View is selected. How these various options work is described below.

 

The top level menu items presented on the Care Checklists main screen may include all or some of the following (depending on access, settings and checklist selected):

Follow:Selecting the Follow button changes the button from "Follow" to "Following". Once a user selects to follow a checklist, this patient's journey through the checklist process will show in the user's home feed. User's also have the option to unfollow a checklist once it is being followed.
Assign Care Steps: nothing about this worked.
Convert to Template: Selecting this option opens the Edit screen which will enable the User to make any changes to the Care Checklist Name and the Care Checklist Type. When the template has been created a positive message will be displayed.
Clone: Selecting this option will clone the existing checklist. Think of this option as a "Save As" option. Simply saving the basic checklist while adding your specific changes. Selecting this option opens a new window to allow for editing of the Care Checklist name and the Type, the system information: Created By and Last Modified By will be updated upon completion of this screen. Completing these field will be described below.
Edit: Selecting this option will allow for the editing of the existing checklist. This option opens a new window to allow for editing of the Care Checklist name and allows for editing the Type it will also update the system information: Created By and Last Modified By. Completing these field will be described below.

Dropdown: This dropdown allows for the option to either delete the selected checklist or change ownership of the selected checklist.

Delete: Selecting this option brings up a second chance dialog box to allow the user to confirm they want to delete the selected checklist.

Change Owner:Selecting Change Owner allows the User to change the ownership of the selected checklist. Once this option is selected a pop up window called Change Owner displays a Search User box as well as a button to allow an email to be sent to the new owner. Enter any system user name into the search box. Select the new owner from the dropdown box. You can then choose to send a notification email. Clicking the SUBMIT button changes ownership, sends the notification email (if that option was selected) and provides a confirmation pop up.

Completing the Edit and Clone Care Checklist Screen

Care Checklist Name: The name of the newly cloned checklist or changes to the existing checklist's name can be entered here.
Type: The Type Dropdown offers two options: Before Care and Post Care. This is a required field.
Created By: This field is system-populated with the UserID of the person who created the checklist. It also provides the date and time the checklist was created.
Last Modified By: This field is system-populated with the UserID of the person who last modified the checklist. It also provides the date and time the checklist was last modified.
Save & New: Selecting this option will save the current changes and open a new dialog box: New Care Checklist Select a Record Type box. This box offers two options: Instance and Template. More information about using this box is found below.
Save: Clicking the save button will save the changes to the new or cloned checklist/template and will provide a confirmation pop up.

Canceled: A Canceled checklist would be a checklist no longer needed by the patient.

Record Type: Two types of records are available: Instance or Template

Instance: Instance types are checklists for a specific patient for a specific care event. A singular event action or series of actions.

Template: Is a template of a care checklist. This would be the basis for a checklist that may be repeated among many patients for a specific care event.
